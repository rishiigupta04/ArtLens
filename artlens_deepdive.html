01 — The problem

What are we actually building, and why?

AI image generators — Midjourney, Stable Diffusion, DALL-E 3 — can now produce artwork that looks indistinguishable from human-made art. This creates real problems:

ARTIST COMMUNITIES

Contest fraud

AI art submitted to human-only contests on ArtStation and DeviantArt. Artists lose to machines they didn't know they were competing against.

STOCK PLATFORMS

Licensing issues

Getty, Shutterstock need to flag AI submissions. Selling AI art under "human-created" licenses is legally and ethically problematic.

ART SCHOOLS

Academic integrity

Students submitting AI art as their own original work. Instructors can't tell by eye anymore.

JOURNALISM

Misinformation

AI-generated images used to fake events, protests, disasters. Fact-checkers need detection tools.

Existing tools like Hive Moderation and AI or Not exist but are paywalled and black-box — they give you a verdict but no explanation. ArtLens is different:

ArtLens goal: Free, explainable, multi-generator detection. Not just "AI or not" — but which generator, why the model thinks so, and how confident it is. Including flagging images from generators it has never seen before.

The three questions ArtLens answers

  1. Is this AI-generated or human-made? — Binary classification (Head 1)
  2. Which generator made it? — Generator fingerprinting (Head 2): Midjourney vs Stable Diffusion
  3. Is this from a generator I've never seen? — Open-set detection (Head 3): flags unknown future generators
02 — Foundations

How computers see images

Before understanding any model, you need to understand what an image actually is to a computer. This is the foundation everything else builds on.

Images are just grids of numbers

A digital image is a 3-dimensional array of numbers. For a 224×224 colour image:

  • Width: 224 pixels
  • Height: 224 pixels
  • Channels: 3 (Red, Green, Blue)
  • Total numbers: 224 × 224 × 3 = 150,528 values

Each number is between 0 and 255. 0 = no intensity, 255 = full intensity. A pure red pixel is [255, 0, 0]. A white pixel is [255, 255, 255]. A black pixel is [0, 0, 0].

A single 224×224 image in PyTorch is a tensor of shape [3, 224, 224] — 3 channels, 224 rows, 224 columns. A batch of 16 images is [16, 3, 224, 224].

Why 224×224?

The Vision Transformer we use (ViT-B/16) was pretrained on ImageNet at 224×224. Using the same size means the pretrained weights apply correctly to our images. All images in our dataset — regardless of original size — are resized to 224×224 before the model sees them.

Normalisation — why we do it

Raw pixel values (0–255) are too large and inconsistent for neural networks to work with efficiently. We normalise every image using the mean and standard deviation of the ImageNet dataset:

# ImageNet statistics — computed over 1.2 million images
mean = [0.485, 0.456, 0.406]  # mean for R, G, B channels
std  = [0.229, 0.224, 0.225]  # std dev for R, G, B channels

# For each pixel: normalised = (pixel/255 - mean) / std
# Result: values centred around 0, typically in range [-2, 2]

Why ImageNet stats specifically? Because ViT-B/16 was pretrained on ImageNet. Using the same normalisation means the model's internal representations from pretraining still apply correctly. Using different stats would be like giving the model images it doesn't recognise.

The frequency domain — a different way to look at images

You can represent any image in two equivalent ways:

Spatial domain (what you see)

Pixel values arranged in a grid. Each value tells you the colour at that exact location. This is the normal "picture" view.

Frequency domain (FFT view)

How fast the pixel values change across the image. Low frequencies = slow changes (backgrounds, broad colours). High frequencies = fast changes (edges, fine textures).

The Fast Fourier Transform (FFT) converts between these two representations mathematically. They contain identical information — just presented differently. The frequency view is useful for ArtLens because:

Key insight: AI generators leave invisible fingerprints in the high-frequency domain. When an AI model upsamples a small feature map to a larger image (a step inside every generator), it creates periodic patterns that appear as regular spikes in the FFT spectrum. Human artists don't create these patterns. The human eye can't see them — but a detector trained on frequency features can.

We also use the Discrete Wavelet Transform (DWT) via PyWavelets. Unlike FFT which analyses the whole image at once, DWT decomposes an image into multiple frequency sub-bands at different scales — giving both frequency and spatial location information simultaneously.

03 — Deep learning basics

Deep learning from scratch

What is a neural network?

A neural network is a function that maps inputs (images) to outputs (labels) by learning from examples. It's made of layers, each of which transforms the data in some way. The "deep" in deep learning just means many layers stacked together.

Think of it like this: you want to teach someone to recognise cats. You can't write rules ("ears pointy, fur fluffy"). Instead, you show them 10,000 cats and 10,000 non-cats, and they learn the pattern. Neural networks do exactly this — but with numbers instead of intuition.

How learning actually works — the training loop

  1. Forward pass: Feed an image through the network, get a prediction (e.g., 73% AI, 27% human)
  2. Loss calculation: Measure how wrong the prediction is. If the image was actually human, the prediction 73% AI is very wrong → high loss
  3. Backward pass (backpropagation): Calculate how much each parameter in the network contributed to the error, using calculus (chain rule)
  4. Update parameters: Nudge each parameter slightly in the direction that reduces the error. This is done by the optimiser (we use AdamW)
  5. Repeat: Do this for thousands of batches across many epochs until the model learns
Analogy: Imagine tuning a radio. You hear static (high loss), you turn the dial slightly (update parameters), check again. You keep adjusting until you find the station (low loss, good predictions). Backpropagation tells you which direction to turn the dial.

Key terms explained

TERM

Epoch

One complete pass through the entire training dataset. We train for 15 epochs — so the model sees every image 15 times.

TERM

Batch size

How many images are processed together before updating parameters. We use batch size 16 — limited by our 6GB GPU.

TERM

Learning rate

How large each parameter update step is. Too high = unstable. Too low = slow. We use 2e-5 — very small for fine-tuning.

TERM

Overfitting

When a model memorises training data but fails on new images. Like a student memorising answers instead of understanding concepts.

TERM

Loss function

The mathematical measure of "how wrong" the model is. We use Cross Entropy Loss — standard for classification tasks.

TERM

Gradient

The direction and magnitude of the slope of the loss function. Tells us which way to nudge each parameter to reduce loss.

What is fine-tuning?

Training a ViT-B/16 from scratch on 30,000 images would be hopeless — it has 86 million parameters and needs millions of images. Instead, we use a model that was already trained on 14 million ImageNet images. It already "knows" what edges, textures, shapes, and objects look like.

Fine-tuning means: start from these pretrained weights, then train further on our specific task (AI art detection) with a very small learning rate. We're nudging an expert, not starting from scratch. This is why we get good results with a relatively small dataset.

04 — Architecture

CNNs vs Vision Transformers

How a CNN works

A Convolutional Neural Network slides a small filter (e.g. 3×3 pixels) across the image, computing a weighted sum at each position. This detects local patterns — edges at specific positions. Stack many such layers and the network builds up from simple edges to complex shapes to semantic concepts.

CNNs are excellent but have a limitation: they're inherently local. A CNN with a 3×3 filter can only see 3×3 pixels at a time. To understand how the top-left corner relates to the bottom-right, the signal has to propagate through many layers — and information can get diluted.

How a Vision Transformer (ViT) works

ViT takes a completely different approach. It slices the image into a grid of non-overlapping patches and treats each patch like a "word" in a sentence. Then it uses the Transformer architecture (originally invented for text) to process all patches simultaneously.

Self-attention — the key mechanism

Self-attention is what makes ViT powerful. For each patch, it computes how much that patch should "pay attention" to every other patch. The result is a weighted average of all patches, where the weights reflect relevance.

Example: when classifying an AI portrait, the patch covering the hair region might attend strongly to the patches covering the facial features, because ViT has learned that inconsistency between these areas is a telltale AI sign. CNNs can't do this directly — they'd need many layers of local convolutions to build up this long-range relationship.

The CLS token: ViT prepends a special learnable vector (called the class token or CLS token) to the sequence of 196 patch embeddings. After all 12 transformer layers, this CLS token has "attended to" all patches and accumulated a global representation of the image. It's this 768-dimensional CLS token vector that we feed into our classification heads.

Why ViT specifically for ArtLens

REASON 1

Global context

AI images often have local inconsistencies — textures that look right individually but wrong together. ViT's self-attention catches this because every patch sees every other patch.

REASON 2

Attention rollout

ViT's attention weights give us explainability for free — we can visualise which patches the model focused on. CNNs need GradCAM bolted on separately.

REASON 3

CLIP connection

Ojha et al. (CVPR 2023) proved CLIP-pretrained ViT features naturally separate real from AI art in embedding space better than any CNN architecture.

05 — Model architecture

ArtLens model — three heads on one backbone

Why three heads instead of three separate models?

Training three separate models would mean three times the computation, three times the VRAM, and three sets of weights to manage. More importantly, the tasks help each other. When Head 2 forces the backbone to learn generator-specific patterns (the subtle fingerprint each generator leaves), those patterns also help Head 1 generalise better. The gradient signals from all three heads flow back into the shared backbone simultaneously, making it learn richer features than any single task would produce.

Head 1 — Binary classification

The simplest head. Takes the 768-dim embedding from ViT, applies LayerNorm, Dropout (30%), then a linear layer producing 2 logits — one for Human, one for AI. Softmax converts these to probabilities. Trained in Phase 2 as a standalone baseline before the other heads are added.

Head 2 — Generator fingerprinting

Only active on AI images (label=1). Classifies whether the AI image came from Midjourney or Stable Diffusion. Each generator has a characteristic "fingerprint" — subtle patterns in how it renders hair, textures, backgrounds. This head forces the backbone to learn these fingerprints. Output: "Midjourney (87% confident)" or "Stable Diffusion (91% confident)".

Head 3 — Open-set detection

This is the research-level contribution. The problem: what if a user uploads an image from DALL-E 3, which the model has never seen during training? A standard classifier would still output "Midjourney" or "Stable Diffusion" — confidently wrong.

Our solution uses Mahalanobis distance. After training, we compute the centroid (average embedding) and covariance matrix of each known generator's cluster in the 768-dim embedding space. At test time, we measure how far the new image's embedding is from all known clusters. If it's far from all of them, it gets flagged as "unknown generator".

Analogy: Imagine you've memorised the handwriting of 5 people. Someone hands you a new letter. You compare it to each person's style. If it doesn't match any of them closely, you say "I don't know who wrote this" — rather than guessing wrong. That's open-set detection.

LightGBM ensemble layer

Parallel to the ViT, we compute handcrafted features from the frequency domain:

  • FFT statistics: High-frequency energy ratio, spectral entropy, peak frequencies — AI images have distinctive patterns here
  • DWT coefficients: Wavelet decomposition across multiple scales — captures texture statistics at different resolutions
  • Local Binary Patterns (LBP): Describes local texture around each pixel — AI art has unnaturally uniform micro-texture

These handcrafted features are concatenated with the 768-dim ViT embedding and fed into LightGBM — a fast, interpretable gradient boosting classifier. If the deep model is fooled by a visually convincing image, the frequency branch may still catch the upsampling artifacts. The two branches act as independent checks on each other.

06 — Dataset design

Why dataset design is 80% of the work

The most common mistake in ML projects is treating the dataset as an afterthought. Every architectural decision we made in the model was motivated by dataset properties. Here's why every choice was made.

Why not use simple photos as the "human" class?

AI generators are trained to produce art, not photographs. A model trained to distinguish photos from AI art would learn "photos = real, paintings = fake" — which is completely wrong. ArtLens needs to work on art images. So our human class is WikiArt — 96,000 images of human-made paintings across 14 art styles.

Stratified sampling — why random isn't good enough

WikiArt is heavily imbalanced: Abstract, Portrait, and Landscape have ~15,000 images each. Marina, Animal Painting, and Flower Painting have ~1,800 each. If we randomly sample 15,000 images, we'd get ~4,000 abstracts and almost no marinas. The model would learn "abstract brushwork = human art" and fail on portrait paintings.

Stratified sampling fixes this: we cap at min(equal_quota, smallest_category) ≈ 1,071 per category and sample exactly that many from each style. Result: every art style contributes equally. Gini coefficient ≈ 0.00 (perfect balance).

The train / val / test split

75% — Training set

What the model learns from. These images are seen repeatedly across all epochs. Never used for evaluation.

15% — Validation set

Used during training to track progress and detect overfitting. Not used to update weights — only to measure performance.

10% — Test set (held out completely) — The model never sees this data until final evaluation after all training decisions are made. This gives us an honest measure of real-world performance. If you evaluate on test data during training, you leak information and fool yourself.

We stratify splits on source (not just label) — ensuring each split contains proportional WikiArt, Midjourney, and SD images. This prevents the pathological case where all Midjourney images end up in only the training set.

Deduplication — two levels

Within-class: pHash (perceptual hashing) finds near-identical images even if slightly resized or re-saved. We keep only one copy per unique pHash.

Cross-class: If the same image appears in both the human and AI sets (rare but catastrophic — the model sees the same image with two different labels), it gets removed from the AI set. This is the most important data quality check.

The CIFAKE trap

Important warning: Many student projects use CIFAKE — a dataset of CIFAR-10 images vs Stable Diffusion v1.4 images. It gives 99%+ accuracy and looks impressive. But SD v1.4 is easy to fingerprint, and the model learns that specific generator's artifacts — not a general "AI art" concept. Interviewers and reviewers will call this out immediately. We deliberately avoid CIFAKE and use JourneyDB (Midjourney v5) and DiffusionDB (Stable Diffusion) instead.
07 — Training

What actually happens during training

fp16 mixed precision — fitting into 6GB VRAM

Your RTX 3050 has 6GB VRAM. ViT-B/16 in full 32-bit precision (fp32) needs ~4GB just for weights. Add gradients (~4GB), activations (~3GB), and optimizer state (~8GB) and you're at ~19GB — way over budget.

Mixed precision (fp16) stores most values in 16-bit floats (half the memory). PyTorch's autocast automatically determines which operations are safe in fp16 and which need to stay in fp32. Combined with GradScaler (which prevents small fp16 gradients from becoming exactly zero), this brings VRAM usage down to ~4GB — fitting on your GPU.

Gradient checkpointing — the other memory trick

During the backward pass, PyTorch needs the activations from every layer to compute gradients. For a 12-layer transformer, storing all these activations takes huge VRAM. Gradient checkpointing says: don't store intermediate activations — just recompute them during the backward pass when needed. This costs ~20% more computation but reduces activation memory by ~60%. One line: model.backbone.set_grad_checkpointing(enable=True).

AdamW optimiser and cosine annealing

We use AdamW — Adam with decoupled weight decay. Adam adapts the learning rate per-parameter based on gradient history (faster learning for rarely-updated parameters). The weight decay (L2 regularisation, 1e-4) prevents weights from growing too large and overfitting.

Cosine annealing gradually reduces the learning rate from 2e-5 to 1e-7 following a cosine curve. This lets the model make larger updates early (exploring the loss landscape) and smaller, more precise updates later (settling into a good minimum). Fixed learning rates are generally worse.

The Wang 2020 augmentation strategy

This is the single most important augmentation decision for generalization. From CNNSpot (Wang et al., CVPR 2020): applying random JPEG compression (quality 50–95) and Gaussian blur to training images dramatically improves the model's ability to detect AI images that have been post-processed.

The reason: when a user uploads an image to Twitter, it gets JPEG-compressed automatically. The high-frequency artifacts that make AI images detectable get partially suppressed. A model trained without this augmentation will fail on compressed images. Training with it forces the model to find more robust, lower-frequency signals instead.

# Wang 2020 augmentation — applied with 50% probability
A.OneOf([
    A.ImageCompression(
        quality_lower=50,
        quality_upper=95   # random JPEG quality
    ),
    A.GaussianBlur(blur_limit=(3, 7))  # random blur kernel
], p=0.5)

Evaluation metrics — why not just accuracy?

METRIC

AUROC

Area Under ROC Curve. Measures how well the model ranks AI images above human images across all confidence thresholds. 0.5 = random, 1.0 = perfect. Threshold-independent — the most honest single metric.

METRIC

F1 Score

Harmonic mean of precision and recall. Accounts for both false positives (calling human art AI) and false negatives (missing AI art). Better than accuracy when classes are imbalanced.

METRIC

Per-source accuracy

How accurate on WikiArt specifically? On Midjourney? On SD? Overall accuracy hides bias. A model 99% on WikiArt and 60% on Midjourney is useless in practice. From the Tasnim 2024 benchmark paper.

METRIC

Confusion matrix

4-cell table: true positives, true negatives, false positives, false negatives. Shows exactly what type of mistakes the model makes. Useful for understanding failure modes before deployment.

Label smoothing — preventing overconfidence

With standard cross-entropy loss, the model is trained to output exactly [0, 1] (100% AI) or [1, 0] (100% human). This pushes the model toward extreme confidence, which is bad for calibration — in the real world, some images genuinely are ambiguous.

Label smoothing (0.1) softens the targets to [0.05, 0.95] and [0.95, 0.05]. The model can never be 100% confident, which produces better-calibrated probabilities in deployment. This directly links to the uncertainty-aware design from the Scientific Reports 2025 paper.

08 — Explainability

Why the model thinks what it thinks

A detector that says "AI" without explanation is nearly useless in practice. Someone disputing a verdict needs to understand which part of the image gave it away. ArtLens uses three complementary explainability methods.

Attention rollout — ViT's native explainability

Each of the 12 transformer layers computes an attention matrix: for each of the 196 patches, how much attention does it pay to every other patch? These matrices are 197×197 (196 patches + 1 CLS token).

Attention rollout recursively multiplies these matrices across all 12 layers, tracking how information flows from the input patches through to the CLS token that makes the final decision. The result is a 14×14 heatmap that we upsample to 224×224 and overlay on the original image.

Bright regions = patches the model "looked at" most = areas that most influenced the prediction. If the heatmap lights up on the hair (AI generators notoriously struggle with hair) or background textures, you can visually explain the decision.

GradCAM++ — gradient-based explanation

GradCAM asks: for the final classification decision, how much did the activations at each spatial location in the last attention layer contribute? It computes the gradient of the class score with respect to the feature maps, uses these gradients to weight the feature maps, and produces a heatmap.

GradCAM and attention rollout capture different things. Attention rollout shows what the model "looked at" throughout all 12 layers. GradCAM shows what was most responsible for the final verdict specifically — more targeted, sometimes more revealing.

SHAP — explaining the LightGBM branch

SHAP (SHapley Additive exPlanations) comes from cooperative game theory. Imagine the features (FFT statistics, wavelet coefficients) as players in a game, and the prediction as the payout. SHAP computes each feature's fair contribution to the prediction, respecting all possible orderings in which features could have been added.

In practice: for each image, SHAP gives you a bar chart showing which frequency features pushed the prediction toward "AI" (positive SHAP value) and which pushed toward "human" (negative). Example output: "High-frequency energy ratio: +0.34 (toward AI), Wavelet detail uniformity: +0.21 (toward AI), LBP variance: -0.08 (toward human)".

Why three methods? Each explains a different part of the model. Attention rollout and GradCAM explain the deep ViT. SHAP explains the classical LightGBM branch. Together they give a complete picture — and showing all three in the UI is a major differentiator from black-box commercial tools.
09 — Research papers

The 5 papers behind ArtLens

Every key architectural decision in ArtLens is grounded in peer-reviewed research. Here's what we take from each paper and where it shows up in the code.

CVPR 2023 Ojha et al.
Towards Universal Fake Image Detectors that Generalize Across Generative Models
Found that standard classifiers fail on unseen generators because they latch onto generator-specific artifacts. Proposed using frozen CLIP-ViT features — which weren't trained for this task — and showed they naturally separate real from AI images better than dedicated detectors. +15 mAP improvement on unseen generators.
↳ Justifies our ViT backbone choice. Motivates the UMAP embedding visualization. Frames the open-set detection problem (Head 3).
CVPR 2020 Wang et al.
CNN-Generated Images Are Surprisingly Easy to Spot... For Now (CNNSpot)
The foundational paper. Showed a ResNet-50 trained on ProGAN generalizes to other generators. Key finding: JPEG compression and Gaussian blur augmentation is essential — without it, detectors fail on post-processed images.
↳ Our augmentation strategy directly copied from this paper. JPEG quality 50–95 + GaussianBlur at p=0.5 in the training transforms.
arXiv 2024 Tasnim et al.
AI-Generated Image Detection: An Empirical Study and Future Research Directions (arXiv:2511.02791)
Benchmarks 10 state-of-the-art methods across 7 datasets. Identifies three critical gaps: non-standardized benchmarks, inconsistent training protocols, limited evaluation metrics. Demonstrates strong in-distribution but degraded cross-generator performance across all methods.
↳ Our evaluation protocol: AUROC + F1 + per-source accuracy breakdown. This is why we report accuracy separately for WikiArt, Midjourney, and SD rather than just overall accuracy.
arXiv 2025 Mahara & Rishe
Methods and Trends in Detecting AI-Generated Images (arXiv:2502.15176)
Comprehensive survey covering FatFormer — a CLIP-based detector combining image and frequency domain features via Discrete Wavelet Transform inside a ViT architecture. Also covers GradCAM-based explainability across multiple methods.
↳ Validates our dual-branch architecture (ViT + frequency features). FatFormer's DWT approach directly inspired our frequency branch. Used as literature review foundation.
Scientific Reports 2025
Detection of AI Generated Images Using Combined Uncertainty Measures and Particle Swarm Optimised Rejection Mechanism
Proposes outputting "uncertain" instead of a confident wrong label when prediction confidence is low, using Monte Carlo Dropout entropy and Gaussian Process variance. Trained on SD, tested on Midjourney, BigGAN, StyleGAN3 — significant distribution shifts.
↳ Justifies our confidence calibration layer (Platt scaling + temperature scaling) in Phase 6. The "reject when uncertain" philosophy is implemented in our open-set head and UI confidence display.

The community feedback loop

When a user disputes a verdict ("I made this, it's real art"), that image is flagged in Supabase. These disputed images become hard negatives — images where the model made a wrong, confident prediction. They automatically queue for the next retraining cycle. The system gets harder to fool over time without any manual data collection.

The batch API

Developers can POST up to 20 images and receive JSON back with: binary label, confidence score, generator guess, and a URL to the attention heatmap. Rate-limited, free tier. This makes ArtLens a tool other developers can integrate, not just a demo website.
